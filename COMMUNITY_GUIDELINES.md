## How to Contribute

### üÜï Getting Started
New to Agentik OS? Start here:

1. **Read the docs** - https://docs.agentik-os.com
2. **Join Discord** - https://discord.gg/agentik-os
3. **Introduce yourself** in #introductions
4. **Find "good first issue"** labels on GitHub
5. **Ask questions** in #help channels

### üíª Code Contributions

**Before you start:**
- Check existing issues/PRs to avoid duplicates
- Discuss major changes in GitHub Discussions first
- Read [CONTRIBUTING.md](./CONTRIBUTING.md) for technical guidelines

**When submitting:**
- Write clear commit messages
- Add tests for new features
- Update documentation
- Follow the code style (ESLint/Prettier)
- Be responsive to review feedback

**PR Checklist:**
- [ ] Tests pass locally (`pnpm test`)
- [ ] TypeScript types are correct (`pnpm type-check`)
- [ ] Documentation updated
- [ ] Changelog entry added (if applicable)
- [ ] Screenshots for UI changes

### üìñ Documentation Contributions

Great docs make great software!

**What we need:**
- Tutorials for common use cases
- Architecture explanations
- Troubleshooting guides
- API reference improvements
- Video tutorials
- Translations

**Tips for writing:**
- Use simple, clear language
- Include code examples
- Add screenshots/diagrams
- Test your tutorial from scratch
- Link to related docs

### üé® Design Contributions

Help make Agentik OS beautiful!

**Design areas:**
- UI/UX for dashboard
- Marketing website design
- Social media graphics
- Documentation visuals
- Logo/branding refinements
- Swag designs

**Design process:**
1. Open issue describing your proposal
2. Share mockups/designs for feedback
3. Iterate based on community input
4. Implement or hand off to developer

### üåê Translation Contributions

Make Agentik OS accessible worldwide!

**Priority languages:**
- Spanish
- French
- German
- Chinese (Simplified & Traditional)
- Japanese
- Korean
- Portuguese (Brazilian)
- Russian

**Translation tools:**
- We use Crowdin for translations
- Contact: i18n@agentik-os.com

### üêõ Bug Reports

Found a bug? Report it!

**Good bug reports include:**
- Clear, descriptive title
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, versions)
- Error messages/logs
- Screenshots if relevant

**Security bugs:**
- **DO NOT** open public issues for security vulnerabilities
- Email: security@agentik-os.com
- Use GitHub Security tab for responsible disclosure

### üí° Feature Requests

Have an idea? Share it!

**Good feature requests:**
- Describe the problem it solves
- Explain the proposed solution
- Consider alternatives
- Note any breaking changes
- Gauge community interest

**After submission:**
- Community discusses and votes
- Core team reviews monthly
- High-value features added to roadmap
- You're welcome to implement it!

## Decision Making

### RFCs (Request for Comments)
Major changes require RFCs:
1. Open GitHub Discussion with [RFC] prefix
2. Community reviews and comments (7-14 days)
3. Core team makes final decision
4. RFC accepted/rejected with reasoning

**What needs RFC:**
- Breaking changes
- New major features
- Architecture changes
- Policy changes

### Community Votes
For non-technical decisions:
- Poll posted in Discord
- Discussion period (3-7 days)
- Voting (3 days)
- Results published

**Voting topics:**
- Event planning
- Swag designs
- Community program changes
- Non-binding product feedback

## Conflict Resolution

### If You Have a Problem

**1. Direct Communication (Preferred)**
- Try resolving directly with the person
- Assume good intent
- Use "I" statements ("I felt..." not "You made me...")
- Focus on behavior, not person

**2. Moderator Mediation**
- If direct resolution fails
- DM any moderator
- They'll help facilitate

**3. Formal Report**
- Use for Code of Conduct violations
- Email: conduct@agentik-os.com
- Or use GitHub Security tab

### If Someone Has a Problem With You

**Stay calm and:**
- Listen without defensiveness
- Ask clarifying questions
- Acknowledge their perspective
- Work toward mutual understanding
- Apologize if appropriate

**Remember:**
- Everyone makes mistakes
- Growth comes from feedback
- Community over ego

## Best Practices

### Communication

**Be Clear:**
- Use simple language
- Provide context
- Include examples
- Check for understanding

**Be Kind:**
- Assume good intent
- Give benefit of doubt
- Praise publicly, criticize privately
- Celebrate others' wins

**Be Constructive:**
- Offer solutions, not just complaints
- Explain the "why" behind feedback
- Be specific, not vague
- Focus on improvement

### Collaboration

**Share Early, Share Often:**
- Don't wait for perfection
- Show work-in-progress
- Ask for feedback
- Iterate together

**Credit Others:**
- Acknowledge contributors
- Link to inspiration sources
- Thank helpers in PR descriptions
- Use Co-authored-by: in commits

**Stay Professional:**
- Keep work hours reasonable
- Respect timezones
- Set boundaries
- Take breaks

## What's Not Okay

### Prohibited Behavior

**Never acceptable:**
- Harassment or discrimination
- Hate speech or slurs
- Doxxing or privacy violations
- Spam or scams
- Impersonation
- Malicious code
- Copyright/license violations

**Context matters:**
- Heated debates ‚Üí OK
- Personal attacks ‚Üí Not OK
- Critical feedback ‚Üí OK
- Mockery/belittling ‚Üí Not OK
- Passionate advocacy ‚Üí OK
- Aggressive pushing ‚Üí Not OK

### Consequences

**First offense:** Warning
**Second offense:** Temporary mute/ban (7 days)
**Third offense:** Permanent ban

**Severe violations** (harassment, doxxing, malicious code) result in immediate permanent ban.
