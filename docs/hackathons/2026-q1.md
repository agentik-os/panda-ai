## Rules

### Project Requirements

**Must:**
1. Use Agentik OS as the core agent runtime
2. Be built during the hackathon week (March 15-22)
3. Have source code on GitHub (public repo)
4. Include README with setup instructions
5. Have a working demo (video or live)
6. Be original work (not a fork of existing project)

**Cannot:**
- Violate Agentik OS license
- Infringe on intellectual property
- Contain malicious code
- Violate any laws
- Promote hate or discrimination

### Submission Requirements

**Required:**
1. GitHub repo link (public)
2. 2-minute demo video (uploaded to YouTube, Loom, or similar)
3. README with:
   - Project description
   - Problem it solves
   - How it works
   - Setup instructions
   - Screenshots
4. List of team members (if team)

**Optional but recommended:**
- Live demo link
- Blog post explaining your project
- Architecture diagram

### Judging Criteria

All projects judged on:
- **Innovation** (30%) - How creative/novel is it?
- **Execution** (25%) - How well is it built?
- **Impact** (25%) - What problem does it solve?
- **Technical Complexity** (20%) - How challenging was it?

Category-specific criteria may apply (see prize sections).

## FAQ

### Can I work on a project I already started?
No. Projects must be started during the hackathon period (March 15-22).

### Can I use external APIs/services?
Yes! Integrate anything you want (OpenAI, Google, Stripe, etc.)

No, but it should have a working demo of the core functionality.
