# Agentik OS Hackathon Rules

Universal rules that apply to all Agentik OS hackathons.

---

## General Rules

### Eligibility

**Who can participate:**
- ✅ Anyone aged 13 or older
- ✅ Solo developers or teams (up to 4 people)
- ✅ Students, professionals, hobbyists
- ✅ Global participation (any country)

**Who cannot participate:**
- ❌ Agentik OS core team members
- ❌ Immediate family of judges
- ❌ Anyone affiliated with prize sponsors
- ❌ Previous hackathon winners (for same category)

### Project Requirements

**Your project must:**
1. **Use Agentik OS** as the primary agent runtime
2. **Be original work** created during the hackathon period
3. **Be open source** (MIT, Apache 2.0, or GPL license)
4. **Have working code** (not just ideas or designs)
5. **Include documentation** (README with setup instructions)

**Your project cannot:**
- Be started before hackathon begins
- Be a fork of existing project (without substantial changes)
- Violate intellectual property rights
- Contain malicious code or security vulnerabilities
- Promote illegal activities or hate speech

## Team Rules

### Team Formation

- **Maximum size**: 4 people
- **Formation**: Before or during hackathon
- **Changes**: Teams can be modified until 48 hours into hackathon
- **Solo allowed**: You don't need a team

### Team Registration

- Register as team lead
- Provide all team member names and emails
- Each member must join Discord #hackathon
- All members must accept rules individually

### Team Collaboration

- **Remote teams**: Allowed (use Discord, Zoom, etc.)
- **Cross-timezone**: Allowed
- **Mentors**: Can advise but not code for you
- **External help**: Stack Overflow, GPT, Copilot are allowed

## Submission Rules

### What to Submit

**Required:**
1. GitHub repository URL (public)
2. Demo video (2-5 minutes)
3. README.md with:
   - Project description
   - Problem statement
   - Solution approach
   - Setup instructions
   - Team members
   - Tech stack
4. License file (MIT, Apache 2.0, or GPL)

**Optional:**
- Live demo link (hosted app)
- Slide deck
- Blog post
- Architecture diagrams
- Additional videos/screenshots

### Submission Deadline

- Submissions due at **23:59 UTC on final day**
- Timezone: UTC (Coordinated Universal Time)
- Late submissions: **Not accepted** under any circumstances
- System time: GitHub commit timestamp is official time

### Submission Platform

Submit through official hackathon portal:
https://hackathon.agentik-os.com

**Backup:** If portal is down, email:
- To: hackathon@agentik-os.com
- Subject: "HACKATHON SUBMISSION - [Team Name]"
- Include all required materials

## Judging Process

### Judging Criteria

All projects judged on (unless category specifies otherwise):

| Criterion | Weight | Description |
|-----------|--------|-------------|
| **Innovation** | 30% | How novel is the idea/implementation? |
| **Execution** | 25% | How well is it built? |
| **Impact** | 25% | Does it solve a real problem? |
| **Technical Merit** | 20% | How complex/impressive is the tech? |

### Judging Panel

- **Core Team** (50% weight) - 3 Agentik OS maintainers
- **Community Vote** (30% weight) - Top 10 projects public vote
- **Guest Judges** (20% weight) - 2-3 industry experts

### Judging Timeline

1. **Submission deadline** - All entries locked
2. **Initial screening** (24h) - Filter incomplete/invalid submissions
3. **Technical review** (48h) - Core team reviews all projects
4. **Community voting** (72h) - Public votes on top 10
5. **Final deliberation** (24h) - Judges discuss and decide
6. **Winners announced** - Live on Discord + community call

### Disqualification

Projects can be disqualified for:
- Missing required submission materials
- Violating any rule
- Plagiarism or code theft
- Malicious code
- Harassment or misconduct
- Late submission

## Prizes

### Prize Distribution

**Eligibility:**
- Must comply with all rules
- Must verify identity (government ID for prizes $200+)
- Must provide payment information
- Must respond within 30 days

**Methods:**
- PayPal (preferred)
- Bank transfer (international)
- Cryptocurrency (BTC, ETH, USDC)
- Check (US only, last resort)

**Timeline:**
- Winners announced: End of judging period
- Contact for verification: Within 24 hours
- Payment processed: Within 30 days of verification
- Tracking provided: For all payments

### Team Prize Split

- Teams decide split amongst themselves
- Notify organizers of split before payment
- Default: Equal split if not specified
- Disputes: Teams must resolve internally

### Tax Implications

- Prizes may be taxable income in your jurisdiction
- Recipients responsible for all taxes
- Form 1099 issued for US winners over $600
- Consult tax professional in your country

### Prize Forfeiture

Prizes forfeited if:
- Winner cannot be contacted within 30 days
- Winner declines prize
- Winner violated rules (discovered post-judging)
- Winner cannot verify identity

Forfeited prizes: Awarded to runner-up in same category

## Fair Play

### Acceptable Practices

✅ **Allowed:**
- Using AI assistants (GPT, Copilot, etc.)
- Searching Stack Overflow
- Reading documentation
- Using code snippets from docs
- Getting general advice from mentors
- Using starter templates we provide

❌ **Not Allowed:**
- Copying another participant's code
- Having someone else code for you
- Starting project before hackathon
- Submitting same project to multiple hackathons
- Creating multiple accounts to vote

### Plagiarism Policy

**Definition:**
- Copying code without attribution
- Submitting others' work as your own
- Using substantial portions of existing projects

**Detection:**
- Code similarity scanning
- Manual review by judges
- Community reporting

**Penalty:**
- Immediate disqualification
- Permanent ban from future hackathons
- Public announcement (if severe)

## Disputes

### Process

1. **Contact organizers:** hackathon@agentik-os.com
2. **Provide evidence:** Screenshots, links, etc.
3. **Investigation:** 48-hour review period
4. **Decision:** Final and binding
5. **Appeal:** One appeal allowed within 24 hours

### Common Disputes

**Code plagiarism:**
- Provide evidence of original work
- Explain any similar code
- Show commit history

**Rule violations:**
- Explain circumstances
- Provide context
- Show good faith effort

**Technical issues:**
- Document the issue
- Show attempted solutions
- Provide timestamps

### Organizer Discretion

Organizers reserve right to:
- Disqualify projects
- Modify rules if necessary
- Extend deadlines in emergencies
- Award additional prizes
- Make final decisions on all matters
