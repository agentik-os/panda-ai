# Community Call Template

Use this template for hosting monthly community calls.

---

## Pre-Call Checklist (1 week before)

### Monday (7 days before)
- [ ] Create agenda discussion thread in Discord #community-calls
- [ ] Pin the agenda thread
- [ ] Announce call date/time in #announcements
- [ ] Tweet about upcoming call
- [ ] Add to community calendar

### Wednesday (5 days before)
- [ ] Review suggested agenda topics
- [ ] Select community showcase (if any)
- [ ] Contact showcase presenter
- [ ] Prepare project update slides

### Friday (2 days before)
- [ ] Finalize agenda
- [ ] Test YouTube streaming setup
- [ ] Create YouTube live event
- [ ] Send reminder in Discord #announcements

### Day Before
- [ ] Reconfirm showcase presenter
- [ ] Test screen sharing
- [ ] Prepare Q&A doc
- [ ] Set up recording

### Day Of (1 hour before)
- [ ] Test audio/video
- [ ] Start YouTube stream (private)
- [ ] Post live stream link in Discord
- [ ] Open voice channel
- [ ] Set up notes doc

## Call Script

### Opening (5 min)

**Host intro:**
```
Welcome everyone to the [Month] Agentik OS Community Call!

I'm [Name], and I'll be your host today.

Quick housekeeping:
- This call is being recorded and will be on YouTube
- Please mute when not speaking
- Use the "Raise Hand" feature for questions
- Chat questions are welcome too

Today's agenda:
1. Project updates
2. Community Q&A
3. [Community showcase, if any]
4. Wrap-up

Let's dive in!
```

### Community Q&A (25 min)

**Managing questions:**
```
Let's open it up for questions. I'll take them from:
1. Discord chat (checking now)
2. Raised hands
3. Pre-submitted questions

First question from the chat: [read question]
```

**If no questions:**
```
While we wait for questions, let me share some common questions
we've been seeing:

Q: [Common question 1]
A: [Answer]

Q: [Common question 2]
A: [Answer]
```

**If too many questions:**
```
Lots of great questions! We'll answer as many as we can.
For questions we don't get to, we'll follow up in Discord.
```

**Technical questions:**
```
Great technical question. Let me share my screen and show you...
[Demo or walk through]
```

---

### Community Showcase (10 min)

**Introducing presenter:**
```
Now for our community showcase!

Today, [Name] is going to show us [brief description].
[Name], thanks for being here. Take it away!
```

**During showcase:**
- Take notes of interesting points
- Prepare 1-2 questions
- Monitor chat for audience questions

**After showcase:**
```
That was awesome, [Name]! Few questions:
- [Question 1]
- [Question 2]
- [From chat] [User] asks: [question]

Where can people find your project?
```

**Wrapping up showcase:**
```
Thanks again, [Name]! Really cool work.
Everyone, check out [link] and give it a star if you found it useful.
```

## During Call - Best Practices

### Managing Energy
- Smile (people can hear it!)
- Vary your tone
- Use natural pauses
- Don't rush

### Managing Time
- Keep intro tight (5 min max)
- Watch the clock during Q&A
- Give 5-min warning before end
- Don't go over time

### Managing Questions
- Repeat question for recording
- Clarify if unclear
- Keep answers concise (2-3 min max)
- "Great question, let's take it to Discord" for deep topics

### Managing Tech Issues
- Have backup host ready
- "We're having technical difficulties, one moment..."
- If audio fails: switch to text-only in Discord
- If can't fix quickly: reschedule

### Managing Difficult Situations

**Someone won't mute:**
- Politely ask: "[Name], could you mute please?"
- Mute them if moderator
- Continue without addressing repeatedly

**Off-topic discussion:**
- "Great discussion! Let's take this to #general after the call"
- "We're getting off-topic, let's refocus on [topic]"

**Inappropriate behavior:**
- "That's not appropriate for this call"
- Mute/remove if necessary
- Report to moderation team

**Dead air/silence:**
- Have filler content ready
- "While we wait for questions, let me show you..."
- Share screen with interesting graph/demo

## Emergency Procedures

### Host Can't Make It
1. Backup host takes over
2. Announce host change at start
3. Follow same script

### Technical Failure
1. Pause 2 minutes to troubleshoot
2. If can't fix: "We're experiencing technical difficulties"
3. Reschedule to same time next day
4. Post in Discord immediately

### No Attendees
1. Wait 10 minutes
2. Record solo update video
3. Post as regular video
4. Reflect on timing/promotion

### Disruptive Participant
1. Warn once
2. Mute if continues
3. Remove if necessary
4. Continue professionally

## Feedback Collection

**Post-call survey (optional 2-min Google Form):**
1. What did you like most about this call?
2. What could be improved?
3. Topics you'd like covered next month?
4. Would you recommend these calls? (Yes/No)
5. Additional comments

**Send survey:**
- Post in Discord #announcements with recording
- Pin for 24 hours
- Review results before next call
